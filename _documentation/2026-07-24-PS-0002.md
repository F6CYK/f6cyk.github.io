# Objet

Ce document constitue la troisième étape du suivi du développement du
menu documentaire récursif. Il fournit un point de reprise complet,
utilisable ultérieurement sans ambiguïté, en décrivant l'état du
système, les contraintes, les limites, les hypothèses validées et le
périmètre des développements restants.

Le fonctionnement général du composant est désormais considéré comme
acquis. Les travaux restants concernent exclusivement la présentation
graphique et la hiérarchie visuelle.

------------------------------------------------------------------------

# Objectif recherché

Le menu documentaire doit :

-   être généré exclusivement à partir des données ;
-   rester totalement récursif ;
-   être indépendant de l'organisation documentaire ;
-   présenter une arborescence verticale claire et lisible ;
-   ne contenir aucune logique de présentation dans les composants
    Liquid.

Ces objectifs sont confirmés et ne doivent plus être modifiés.

------------------------------------------------------------------------

# Organisation des fichiers

-   `_data/documentation.yml`
-   `_includes/nav.html`
-   `_includes/recursive-menu.html`
-   `assets/css/layout.css`
-   `assets/css/components.css`
-   `assets/img/`

Aucune modification d'architecture n'est envisagée.

------------------------------------------------------------------------

# État d'avancement

Les éléments suivants sont définitivement acquis :

-   modèle documentaire récursif ;
-   génération Liquid fonctionnelle ;
-   intégration dans la navigation principale ;
-   ouverture récursive des sous-menus ;
-   suppression du comportement horizontal ;
-   stabilisation du fonctionnement en **version v4** ;
-   absence de régression dans les composants Liquid.

Le développement ne porte plus sur ces aspects.

# Fonctionnement actuel

La **version v4** assure :

-   une ouverture verticale correcte ;
-   une récursivité complète ;
-   une disparition des cascades horizontales ;
-   une disparition des boîtes imbriquées ;
-   une compatibilité totale avec l'architecture documentaire.

Le fonctionnement est considéré comme validé.

------------------------------------------------------------------------

# État actuel de la présentation

La **version v4** présente :

-   une ouverture correcte des niveaux ;
-   une structure récursive lisible ;
-   la disparition des blocs horizontaux.

## Limites actuelles

-   hiérarchie visuelle insuffisamment marquée ;
-   niveaux affichés dans un bloc unique ;
-   absence de différenciation graphique ;
-   profondeur non perceptible.

Ces limites constituent désormais le périmètre exclusif du
développement.

------------------------------------------------------------------------

# Choix d'architecture validés

-   aucune modification des données ;
-   aucune modification des composants Liquid ;
-   aucune modification de `recursive-menu.html` ;
-   développement exclusivement réalisé dans `assets/css/components.css`
    ;
-   séparation stricte entre fonctionnement et présentation ;
-   interdiction d'introduire une logique CSS dépendante de
    l'organisation documentaire.

Ces choix sont définitifs.

------------------------------------------------------------------------

# Retour d'expérience

Les travaux ont montré que :

-   modifier plusieurs règles CSS simultanément conduit à des
    régressions ;
-   le délai de publication GitHub Pages (10 à 11 minutes) impose une
    discipline stricte ;
-   les essais parallèles ou non documentés entraînent une perte de
    contrôle ;
-   la **version v4** est le seul point stable identifié.

Ces constats imposent une méthode rigoureuse.

------------------------------------------------------------------------

# Enseignements de la session

-   une seule version de référence du fichier `components.css` ;
-   interdiction de repartir d'une version antérieure ;
-   une seule modification CSS par itération ;
-   validation obligatoire après publication ;
-   aucune expérimentation parallèle ;
-   aucune remise en cause de l'architecture.

Ces règles constituent le protocole de développement.

------------------------------------------------------------------------

# Point d'arrêt

**Version de référence : v4**

Le développement est interrompu à la fin de cette version. Toute reprise
devra impérativement s'appuyer sur cette base.

# Affichage des images

## Emplacement

Les fichiers image doivent être placés dans :

``` text
assets/img/
```

Exemples :

-   `assets/img/xr2206.jpg`
-   `assets/img/composant.png`
-   `assets/img/schema.png`

## Déclaration dans `_data/documentation.yml`

Seul le nom du fichier doit être indiqué :

``` yaml
image: xr2206.jpg
```

Aucun chemin ni sous-répertoire.

## Génération du chemin Liquid

``` liquid
{{ '/assets/img/' | append: item.image | relative_url }}
```

Ce mécanisme garantit la compatibilité avec GitHub Pages.

## Intégration dans `recursive-menu.html`

``` html
<img class="doc-image"
     src="{{ '/assets/img/' | append: item.image | relative_url }}"
     alt="{{ item.title }}">
```

## Style CSS

``` css
.doc-image {
    max-width: 100%;
    height: auto;
    margin: 0.5rem 0;
}
```

## Résultat

Les images s'affichent correctement :

-   à tous les niveaux du menu récursif ;
-   sans rupture ;
-   sans dépendance à la profondeur ;
-   sans logique conditionnelle supplémentaire.

Cette section fait désormais partie intégrante du protocole de
développement.
