# Conventions de nommage

| Document | Conventions de nommage |
|----------|-------------------------|
| Version | 1.0 |
| Statut | En vigueur |
| Date | 22 juillet 2026 |

---

# 1. Objet

Le présent document définit les conventions de nommage utilisées dans le projet F6CYK.

Ces conventions ont pour objectif :

- d'assurer la cohérence de l'ensemble du site ;
- de faciliter la maintenance ;
- de garantir la pérennité des contenus ;
- d'obtenir des noms lisibles et explicites.

---

# 2. Principes généraux

Les noms doivent être :

- explicites ;
- concis ;
- stables dans le temps.

Ils doivent permettre d'identifier immédiatement la nature de la ressource.

---

# 3. Jeux de caractères

Les noms utilisent exclusivement :

- les lettres minuscules de l'alphabet latin (`a` à `z`) ;
- les chiffres (`0` à `9`) ;
- le tiret (`-`) comme séparateur.

Les caractères suivants sont proscrits :

- espaces ;
- lettres accentuées ;
- apostrophes ;
- guillemets ;
- parenthèses ;
- caractères spéciaux.

---

# 4. Répertoires

Les répertoires portent des noms courts et représentatifs de leur contenu.

Exemples :

```text
assets/
documents/
pages/
_posts/
_documentation/
```

---

# 5. Articles

Le nom d'un article doit résumer clairement son sujet.

Exemples :

```text
antenne-levy.md
bobinage-nid-abeille.md
xr2206.md
```

---

# 6. Répertoires de ressources

Les ressources d'un article sont regroupées dans un répertoire portant le même nom que l'article.

Exemple :

```text
xr2206/
antenne-levy/
bobinage-nid-abeille/
```

---

# 7. Images

Les images sont numérotées dans leur ordre d'utilisation.

Exemples :

```text
01.jpg
02.jpg
03.jpg
schema-principe.svg
schema-electrique.svg
implantation.png
```

Le nom doit décrire le contenu lorsque la simple numérotation n'est pas suffisante.

---

# 8. Documents

Les documents conservent autant que possible leur nom d'origine lorsqu'il présente un intérêt documentaire.

Dans le cas contraire, un nom explicite est préféré.

Exemples :

```text
manuel-xr2206.pdf
datasheet-ne602.pdf
radio-ref-1978-03.pdf
```

---

# 9. Audio

Les fichiers audio portent un nom décrivant leur contenu.

Exemples :

```text
qso-am-80m.mp3
cw-14mhz.mp3
ft8-20m.mp3
bruit-bande-160m.mp3
```

---

# 10. Vidéo

Les vidéos sont nommées selon leur sujet.

Exemples :

```text
presentation-xr2206.mp4
construction-antenne-levy.mp4
field-day-2026.mp4
```

---

# 11. Évolution

Les noms de fichiers et de répertoires ne doivent être modifiés qu'en cas de nécessité.

Un changement de nom peut entraîner des liens rompus et compliquer la maintenance.

La stabilité des noms constitue un principe important de l'organisation du site.

---

# 12. Principe directeur

Un nom doit être compréhensible sans avoir à ouvrir le fichier correspondant.

Il doit permettre d'identifier immédiatement la nature et le contenu de la ressource.
